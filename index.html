Introduction

resty-vmadm provides a thin JSON+REST API shim around SmartOS's system utilities so you can programatically manage Zones & Virtual Machines as well as monitor system capacity and health.

Goals

  1. Self-contained
    No external dependencies in the global zone.
  2. Limited scope
    Provide a means for other tools to manage a node.
    Make higher-level tools possible. Don't be the high-level control panel.

Installation & Configuration

On a SmartOS global zone
  1. Copy the resty-vmadm source directory to /opt/resty-vmadm
  2. Link SMF manifest in for autostart:
    ln -s /opt/resty-vmadm/resty-vmadm.xml /opt/custom/smf/resty-vmadm.xml
  3. Edit the config to set port numbers, username, password, etc in:
    /opt/resty-vmadm/etc/resty-vmadm.json
  4. reboot the system to make sure everything comes up cleanly

Security note

resty-vmadm is pretty dangerous and should only be run in a trusted environment. Your global zone is on a restricted management network, right? Seriously, don't expose this to anything you don't trust. It runs as root and has minimal(no) data validation.

There is optional HTTP Basic authentication (if you set a username & password in the config file). This is to help prevent mistakes on your management network, not to secure it so you can expose it to the world.
